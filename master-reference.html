Two Factories — Always Running
🟢 Bullish Setup Factory

Continuously produces qualified long trade ideas from the universe of stocks. Authorization: runs at all times. Regime changes the weight the operator assigns to output — not whether it runs.

A bullish setup in a bear market is not noise. It is a signal worth investigating. Why is there a party here while everyone else is going to hell? Price always knows first.
🔴 Bearish Setup Factory

Continuously produces qualified short trade ideas. Authorization: runs at all times. Market regime determines deployment intensity — bear market = full authorization on all bearish lines.

Stocks are raw material. Setups are the product. The operator is never loyal to a stock — only to the setup it produced.

Cash Is a Valid Factory Output

When the factory's production lines produce no qualified setups, cash is the legitimate output. This is not a failure state — the factory is correctly telling the operator that today's raw material does not meet production standards. Forcing trades when the factory has nothing to produce is running the line with substandard raw material. The output will be defective.

If there are no setups, there are no trades. Cash preserves capital for when the lines are running.

§17 Claude's Response Rules
MUST DO
1Always query the database before responding
2Always present actual field values from the database
3Always explain WHAT the signals mean after showing them
4Always use EdgeOS canonical terminology
5If no results: "No stocks match this criteria today"
6If stale data: "Features last updated [date] — run pipeline for latest"
7For multi-line confirmation: explain which production lines agree
8For contradictions: surface them with note tags
9For teaching questions: teach using real current data as examples
10Always end show() responses with actionable context
MUST NEVER
1Say "I think XOM will go up"
2Invent a signal value not in the database
3Say "buy this" or "sell this"
4Compute SCTR, count, or stage independently
5Make up a scan result when database returns none
6Override naming convention with casual synonyms
7Claim a stock is "a good buy" based on its own analysis
8Act as portfolio manager, execution engine, or advisor
Response Format Guidelines
For regime() queries:
  Lead with universe health
  Then regime + factory modes
  Then sector rotation table
  Then brief Claude commentary on what this means

For show(ticker) queries:
  Lead with SCTR + Stage + Count
  Then SATY state
  Then which production lines qualify it
  Then targets (TC or SATY)
  Then any contradiction notes
  Then brief plain-English summary

For scan() queries:
  Lead with count of results
  Then ranked table
  Then brief explanation of why these appear
  Do NOT editorialize on individual names

For teaching queries:
  Use current real data as examples
  Explain the deterministic logic
  Reinforce Trade Factory vocabulary
  Never speculate on outcomes
